# 검증 계획서

## 1. 문서 목적

이 문서는 `kiwoom_rest_mock_quant_autotrading_plan.md` 기획서와 `development_spec.md` 개발 명세서를 기반으로, 시스템의 정확성과 안정성을 검증하기 위한 계획을 정의한다.  
단위 테스트, 통합 테스트, 모의투자 전진 검증, 백테스트 검증, 실전 전환 체크리스트를 포함한다.

## 2. 검증 단계 개요

```
단위 테스트 → 통합 테스트 → 백테스트 검증 → 모의투자 전진 검증 → 실전 전환 체크리스트
```

| 단계 | 목적 | 환경 | 판정 주체 |
|---|---|---|---|
| 단위 테스트 | 개별 함수/클래스 정확성 | 로컬, mock 데이터 | 자동 (pytest) |
| 통합 테스트 | 모듈 간 연동, 실제 API 호출 | 모의투자 API | 자동 + 수동 확인 |
| 백테스트 검증 | 전략 성과 및 엔진 신뢰성 | 과거 데이터 | 수동 분석 |
| 모의투자 전진 검증 | 실시간 환경에서 전략/시스템 안정성 | 모의투자 실시간 | 일간/주간 리뷰 |
| 실전 전환 체크리스트 | 실거래 전환 준비 완료 확인 | - | 수동 점검 |

## 5. 백테스트 검증

### 5.1 검증 목적

백테스트 엔진 자체의 신뢰성과 전략 성과를 분리해서 검증한다.

### 5.2 엔진 신뢰성 검증

백테스트 엔진이 정확하게 동작하는지를 알려진 데이터로 확인한다.

#### 검증 A. 체결 로직 정확도

```
테스트 데이터: 3일분 5분봉 (수동 제작, 가격 패턴 알려진 상태)

케이스 1: 시장가 매수
  - 시그널: 10:00 봉에서 ENTRY
  - 기대 체결: 10:05 봉 시가 + 슬리피지(0.1%)
  - 검증: 체결가 == 10:05 시가 * 1.001

케이스 2: 지정가 매수 (체결)
  - 시그널: 10:00 봉에서 ENTRY, 지정가 10,000원
  - 10:05 봉 저가 = 9,950원 (지정가 이하)
  - 기대: 체결, 체결가 = 10,000원
  - 검증: 체결 확인, 체결가 정확

케이스 3: 지정가 매수 (미체결)
  - 시그널: 10:00 봉에서 ENTRY, 지정가 9,500원
  - 10:05 봉 저가 = 9,600원 (지정가 미도달)
  - 기대: 미체결
  - 검증: 주문 상태 = pending

케이스 4: 손절
  - 진입가 10,000원, 손절 -3% = 9,700원
  - 특정 봉 저가가 9,650원
  - 기대: EXIT 시그널 발생, 다음 봉 시가로 매도
  - 검증: 청산 확인, 실현손익 정확
```

#### 검증 B. 리샘플링 정확도

```
테스트 데이터: 1거래일 390개 1분봉

검증:
- 5분봉 리샘플링 → 78개 봉
- 각 5분봉의 Open == 첫 1분봉의 Open
- 각 5분봉의 High == 5개 1분봉 중 max(High)
- 각 5분봉의 Low == 5개 1분봉 중 min(Low)
- 각 5분봉의 Close == 마지막 1분봉의 Close
- 각 5분봉의 Volume == 5개 1분봉 Volume 합산

경계 케이스:
- 장 시작 09:00 첫 봉 처리
- 장 마감 15:20 마지막 봉 절단 처리
- 반일 거래일 12:30 마감 봉 처리
```

#### 검증 C. 성과 지표 정확도

```
테스트 데이터: 수동으로 계산한 10건의 거래 (진입/청산 가격, 수수료 확정)

검증 항목:
- 총 수익률 = 수동 계산값 (오차 < 0.01%)
- MDD = 수동 계산값 (오차 < 0.01%)
- 승률 = 수동 계산값 (정확히 일치)
- Profit Factor = 수동 계산값 (오차 < 0.01%)
```

### 5.3 전략 성과 검증

#### 검증 대상 기간

- **최소 6개월**: 과거 일봉 기반 기본 검증 (분봉 데이터 부족 시)
- **최소 3개월**: 분봉 기반 정밀 검증 (분봉 데이터 축적 후)
- **시장 국면별**: 상승장, 하락장, 횡보장 구간을 각각 포함

#### 전략별 통과 기준

| 지표 | 기준 | 비고 |
|---|---|---|
| MDD | < 10% | 초과 시 파라미터 재조정 |
| 승률 | > 40% | 전략 유형별 차등 가능 |
| Profit Factor | > 1.3 | 수수료/슬리피지 반영 후 |
| 월간 손실 | 2개월 연속 적자 없음 | 3개월 이상 백테스트 시 |
| 일간 최대 손실 | < 3% | 단일 거래일 기준 |

#### 슬리피지 민감도 분석

동일 전략을 슬리피지 가정치를 바꿔가며 실행해 성과 변화를 확인한다.

| 슬리피지 | CAGR | MDD | Profit Factor |
|---|---|---|---|
| 0.00% | (측정) | (측정) | (측정) |
| 0.05% | (측정) | (측정) | (측정) |
| 0.10% | (측정) | (측정) | (측정) |
| 0.20% | (측정) | (측정) | (측정) |
| 0.30% | (측정) | (측정) | (측정) |

**통과 기준**: 슬리피지 0.20%에서도 Profit Factor > 1.0

#### 워크포워드 테스트

과적합 방지를 위해 아래 절차로 실행한다.

```
전체 기간: 6개월

1차: 학습 4개월 → 검증 2개월
2차: 학습 기간을 1개월 전진 → 검증 1개월 전진
3차: 반복

통과 기준:
- 검증 구간 성과가 학습 구간 대비 50% 이상 유지
- 검증 구간에서 Profit Factor > 1.0
- 3회 이상 반복 중 2회 이상 통과
```

#### 벤치마크 비교

모든 백테스트 결과에 아래 벤치마크 대비 성과를 포함한다.

- KOSPI 지수 동일 기간 수익률
- KOSDAQ 지수 동일 기간 수익률
- KODEX 200 ETF 동일 기간 수익률

**통과 기준**: 최소 1개 벤치마크 대비 초과수익률 양수

### 5.4 백테스트 결과 리포트 양식

```
=== 백테스트 결과 ===

```

## 6. 모의투자 전진 검증

### 6.1 목적

백테스트에서는 발견되지 않는 실시간 환경의 문제를 검출한다.

- API 호출 안정성
- 체결 타이밍/슬리피지 현실성
- 스케줄러 정확도
- 장애 복구 동작
- 리스크 관리 실효성

### 6.2 검증 기간

- **최소 4주** (20거래일)
- 가능하면 **8주** 권장 (다양한 시장 국면 경험)

### 6.3 일간 점검 항목

매 거래일 장 종료 후 아래 항목을 점검한다.

```
[ ] 봇이 장전~장후까지 정상 동작했는가
[ ] API 에러/타임아웃 발생 건수
[ ] 토큰 갱신 실패 여부
[ ] Rate Limit 초과 발생 여부
[ ] 스캔 주기가 정확히 5분 간격인가
[ ] 미체결 주문 처리 (정정/취소) 정상 동작
[ ] 손절/익절 트리거 정상 동작
[ ] 텔레그램 알림 발송 정상
[ ] 일간 성과리뷰 자동 생성 확인
[ ] DB에 로그/주문/체결 정상 저장
```

### 6.4 주간 점검 항목

매주 마지막 거래일에 아래 항목을 추가 점검한다.

```
[ ] 주간 누적 수익률 확인
[ ] 주간 MDD 확인 (5% 미만 유지)
[ ] 주간 성과리뷰 자동 생성 확인
[ ] 백테스트 대비 성과 괴리 범위 확인
[ ] 슬리피지 실측치와 백테스트 가정치 비교
[ ] 미체결 비율 확인
[ ] 체결 소요 시간 통계 확인
[ ] 전략별 시그널 발생 빈도 적정 여부
[ ] 유니버스 필터링 결과 적정 여부 (종목 수 너무 적거나 많지 않은지)
```

### 6.5 전진 검증 통과 기준

| 항목 | 기준 |
|---|---|
| 시스템 가동률 | 거래일 기준 95% 이상 (장중 정상 가동) |
| API 에러율 | 전체 호출 대비 1% 미만 |
| 장애 복구 | 발생 시 5분 이내 자동 복구 |
| MDD | 4주간 누적 7% 미만 |
| 백테스트 괴리 | 수익률 괴리 절대값 < 백테스트 수익률의 50% |
| 슬리피지 실측 | 평균 슬리피지 < 0.20% |
| 미체결 비율 | 지정가 주문 미체결 < 30% |
| 리스크 트리거 | 손절/익절이 의도한 가격 범위 내에서 동작 |

### 6.6 전진 검증 실패 시 대응

| 실패 유형 | 대응 |
|---|---|
| 시스템 장애 | 원인 분석 → 수정 → 2주 재검증 |
| 전략 성과 부진 | 파라미터 재조정 → 백테스트 재실행 → 2주 재검증 |
| 슬리피지 과대 | 체결 방식 변경 (시장가 → 지정가) 또는 유니버스 유동성 기준 강화 |
| 백테스트 괴리 과대 | 백테스트 슬리피지/체결 모델 보정 후 재실행 |

## 7. 실전 전환 체크리스트

모의투자 전진 검증을 통과한 후, 실전 전환 전에 아래 항목을 모두 확인한다.

### 7.1 시스템 준비

```
[ ] 실거래 도메인(api.kiwoom.com) 연결 테스트 통과
[ ] 실거래 계좌 API 키 발급 완료
[ ] 실거래용 .env 파일 분리 (모의투자 설정과 혼동 방지)
[ ] 설정에서 is_mock = False 전환 시 도메인 자동 변경 확인
[ ] 실거래 계좌 예수금 조회 성공
[ ] 실거래 계좌 잔고 조회 성공
```

### 7.2 리스크 강화

```
[ ] 실전용 리스크 파라미터 별도 설정
    - 1회 주문 최대 비중: 10% (모의투자 15%에서 축소)
    - 1일 최대 손실: 1.5% (모의투자 2%에서 축소)
    - 최대 포지션 수: 3개 (모의투자 5개에서 축소)
[ ] Kill Switch 동작 재확인
[ ] 드로다운 비중 축소 임계치 재확인
[ ] 연속 손실 3회 시 전략 자동 정지 확인
```

### 7.3 안전장치

```
[ ] 첫 1주일은 최소 금액(100만원 이하)으로 운영
[ ] 첫 2주일은 1일 1~2회만 매매 (자동 진입 횟수 제한)
[ ] 텔레그램 알림 전 등급 활성화 (INFO 포함)
[ ] 실전 첫 주문은 수동 확인 후 실행 (자동매매 시작 전 1회 수동 주문)
```

### 7.4 모니터링

```
[ ] 실전 첫 3거래일은 장중 상시 모니터링
[ ] 헬스체크 5분 주기 자동 확인
[ ] 슬리피지 실측치 일일 기록
[ ] 백테스트 대비 성과 괴리 일일 기록
```

### 7.5 롤백 기준

아래 중 하나라도 해당되면 즉시 봇 정지 후 모의투자로 복귀한다.

```
- 1일 손실 > 1.5%
- 3거래일 누적 손실 > 3%
- 연속 5회 손실
- API 에러로 인한 주문 실패 2회 이상 / 1일
- 미체결 주문이 30분 이상 방치
- 의도하지 않은 종목/수량으로 주문 발생
```
