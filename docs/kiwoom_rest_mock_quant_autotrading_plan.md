# 키움증권 REST API 기반 모의투자 자동매매 기획안

## 1. 문서 목적

이 문서는 키움증권 OpenAPI의 REST API를 활용해 국내 주식 모의투자용 자동매매 프로그램을 구축하기 위한 기획안이다.  
목표는 다음 3가지를 동시에 만족하는 것이다.

1. 키움 REST API 기준으로 실제 구현 가능한 구조를 잡는다.
2. 퀀트 기반 전략 5종을 설계하고, 사용자가 전략을 선택해 실행할 수 있게 한다.
3. 이후 실거래 전환, 전략 추가, 백테스트 고도화가 쉬운 형태로 시스템 기본 골격을 만든다.

## 2. 공식 문서 기준 전제

키움 공식 REST API 가이드 기준으로 확인한 핵심 사항은 아래와 같다.

- OAuth 토큰 발급은 `POST /oauth2/token` 이다.
- 운영 도메인은 `https://api.kiwoom.com` 이고, 모의투자 도메인은 `https://mockapi.kiwoom.com` 이다.
- 모의투자는 `KRX만 지원가능` 하다.
- 국내주식 REST 카테고리로 계좌, 시세, 차트, 조건검색, 주문 API가 제공된다.
- 국내주식 주문은 `POST /api/dostk/ordr` 이고, 대표 TR로 `kt10000`(주식 매수주문), `kt10001`(주식 매도주문), `kt10002`(정정), `kt10003`(취소)가 제공된다.
- 차트 조회는 `POST /api/dostk/chart` 이고, 대표 TR로 `ka10080`(분봉), `ka10081`(일봉) 등이 제공된다.
- 계좌 조회는 `계좌평가잔고내역요청(kt00017)`, `예수금상세현황요청(ka10170)`, `미체결요청(ka10075)` 등을 사용할 수 있다.
- 실시간 시세/체결/잔고는 WebSocket도 제공된다.
- 조건검색 목록 및 실시간 조건검색은 WebSocket 기반 항목이 제공되며, 조건검색식은 영웅문4에서 사전 생성해야 한다.
- REST API는 초당/분당 호출 수 제한(Rate Limit)이 있으므로 호출 빈도 관리가 필수다.

참고:

- 공식 가이드: https://openapi.kiwoom.com/guide/apiguide?dummyVal=0
- 모바일 가이드 검색 결과 기준 확인 항목: 주문 `kt10000`, `kt10001`, 차트 `ka10080`, `ka10081`, 계좌 `kt00017`, `ka10170`, `ka10075`, 조건검색 `ka10171`

## 3. 개발 방향 요약

### 추천 방향

이번 1차 구축은 `REST 중심 하이브리드 구조`를 추천한다.

- 기본 조회, 주문, 잔고, 전략 계산은 REST API로 처리
- 실시간 체결 감시와 장중 가격 반응성 향상을 위해 WebSocket은 선택적으로 사용
- 모의투자 초기 버전은 일단 REST만으로도 동작 가능하게 설계
- 다만 실전성과 안정성을 위해 2차 단계에서 WebSocket 보강을 전제로 한다

즉, 사용자가 원한 "REST API로 동작하는 기본"은 만족하되, 자동매매 특성상 완전 REST-only는 반응 속도와 주문 관리 측면에서 불리하므로 구조는 처음부터 WebSocket 확장 가능하게 잡는다.

### 백테스트 방향

백테스트는 `일봉 전용`이 아니라 `분봉 기반 멀티전략 백테스트 엔진`을 전제로 설계해야 한다.

- 전략별로 사용하는 봉 주기가 다를 수 있으므로 1분봉 원천 데이터를 기준으로 저장한다.
- 실행 엔진은 1분봉을 받아 3분봉, 5분봉, 10분봉, 15분봉으로 재샘플링 가능해야 한다.
- 전략은 동일 인터페이스를 사용하되, 각 전략이 원하는 타임프레임을 선언할 수 있어야 한다.
- 백테스트와 실거래 엔진이 같은 시그널 생성 로직을 재사용하도록 설계한다.
- 일봉은 보조 지표 계산용으로만 쓰고, 실제 진입/청산 검증은 분봉 기준으로 수행한다.

## 4. 시스템 목표

### 1차 목표

- 모의투자 계좌 연동
- 국내장 자동매매 봇 실행
- 전략 5종 탑재
- 전략 선택형 실행
- 텔레그램 봇을 통한 전략 선택 및 운영 제어
- 텔레그램 매수/매도/오류 알림
- 종목 스크리닝 -> 매수 후보 선정 -> 주문 -> 포지션 관리 -> 매도까지 자동화
- 로그, 주문 이력, 성과 요약 저장

### 2차 목표

- 백테스트 엔진
- 파라미터 최적화
- 실시간 조건검색 연동
- 슬리피지/수수료 반영 강화
- 일/주/월 성과리뷰 자동 발행
- 실거래 전환용 안전장치

## 5. 추천 자동매매 운영 방식

국내장 기준으로는 초기에 아래 방식이 가장 현실적이다.

### 추천 방식: 일중 1~3회 리밸런싱 + 종가/장중 혼합형

- 장 시작 직후 9:00~9:10은 변동성이 매우 커서 추격 진입 위험이 높다.
- 초반 5~15분 데이터를 본 뒤 진입하는 방식이 안정적이다.
- 포지션 수는 3~8개로 제한한다.
- 종목당 투자 비중은 동일가중 또는 변동성 역가중을 추천한다.
- 손절/익절/시간청산 규칙을 반드시 둔다.
- 첫 버전은 초단타보다 `단기 스윙 + 당일 관리형`이 구현 난이도와 안정성 면에서 유리하다.

### 추천 주문 정책

- 진입: 지정가 우선, 유동성이 충분할 때만 시장가 또는 최유리 사용
- 청산: 손절은 신속성 우선, 익절은 분할청산 허용
- 주문 전 가용현금/주문가능수량 점검
- 미체결 발생 시 정정/취소 로직 필수

### 추천 리스크 정책

- 1회 주문 최대 비중: 총 자산의 15~20%
- 1일 총 손실 한도: 계좌 평가자산의 2%
- 종목당 손절: -2% ~ -4%
- 일일 최대 신규 진입 횟수 제한
- 장 종료 전 미청산 허용 여부를 전략별로 분리

## 6. 전략 구성안

전략은 모두 국내 주식 모의투자 기준으로 설계하며, 첫 버전에서는 `전략 인터페이스`를 통일해 사용자가 하나를 선택해서 돌릴 수 있게 만든다.

---

### 전략 1. 단기 모멘텀 돌파 전략

#### 개요

전일 고가/최근 N일 박스권 상단 돌파 종목을 매수하는 전략이다.  
국내장에서 거래대금이 붙는 강한 종목을 추종하는 방식으로, 구현이 비교적 단순하고 실전성이 높다.

#### 진입 조건 예시

- 최근 20일 평균 거래대금 이상
- 당일 거래대금 상위권
- 현재가가 최근 20일 고가 또는 직전 박스 상단 돌파
- 시가 대비 지나친 갭 상승 종목은 제외

#### 청산 조건 예시

- 손절: 진입가 대비 -3%
- 익절: +5% 또는 추세 이탈
- 시간청산: 보유 3거래일 초과 시 정리

#### 장점

- 강한 종목을 따라가므로 상승장 적응력이 좋다.
- REST 기반 일봉/분봉 + 시세 데이터로 구현 가능하다.

#### 단점

- 횡보장에서 손절 빈도가 높을 수 있다.

---

### 전략 2. 눌림목 추세추종 전략

#### 개요

중기 상승 추세 종목이 단기 조정을 거친 뒤 다시 반등하는 지점을 노린다.

#### 진입 조건 예시

- 20일 이동평균선 > 60일 이동평균선
- 최근 60일 신고가 근처 이력 존재
- 최근 3~5일 조정 후 거래량 감소
- 당일 반등 캔들 또는 5일선 회복

#### 청산 조건 예시

- 손절: 20일선 이탈 또는 -3%
- 익절: 전고점 근처 분할매도
- 추세 훼손 시 전량 청산

#### 장점

- 돌파 추격보다 진입단가가 유리하다.
- 단기 과열 회피에 유리하다.

#### 단점

- 강한 상승 초입을 놓칠 수 있다.

---

### 전략 3. 평균회귀 과매도 반등 전략

#### 개요

급락 후 기술적 반등 가능성이 높은 종목을 짧게 매매한다.

#### 진입 조건 예시

- RSI(2) 또는 RSI(5) 과매도
- 최근 3~5일 낙폭 과대
- 거래대금이 너무 낮지 않을 것
- 하한가 근접, 관리종목, 급격한 악재 종목 제외

#### 청산 조건 예시

- 익절: +2% ~ +4% 단기 청산
- 손절: -2%
- 당일 또는 익일 시가 기준 시간청산

#### 장점

- 횡보장에서도 기회가 생길 수 있다.
- 보유 기간이 짧아 리스크 관리가 명확하다.

#### 단점

- 하락추세 종목에 역추세 진입하는 위험이 있다.

---

### 전략 4. 저변동성 추세 지속 전략

#### 개요

급등주보다 조용하게 우상향하는 종목을 선별해 추세를 따라가는 전략이다.

#### 진입 조건 예시

- 최근 60일 수익률 양수
- 변동성은 시장 평균 이하
- 20일/60일 이동평균 우상향
- 거래대금 최소 기준 충족

#### 청산 조건 예시

- 20일선 이탈
- 변동성 급등 및 추세 훼손
- 보유 기간 10~20거래일 도달 시 재평가

#### 장점

- 상대적으로 매매 스트레스가 낮다.
- 단기 잡음이 적어 자동화에 유리하다.

#### 단점

- 급등장에서 수익 속도가 느릴 수 있다.

---

### 전략 5. 멀티팩터 랭킹 전략

#### 개요

모멘텀, 거래대금, 변동성, 추세, 낙폭 회복력 등 여러 요소를 점수화해 상위 종목을 매수한다.

#### 팩터 예시

- 20일 수익률 점수
- 60일 수익률 점수
- 최근 거래대금 점수
- 추세 정배열 점수
- 변동성 패널티
- 과도한 갭 상승 패널티

#### 운영 방식

- 매일 장 시작 전 또는 장중 1회 랭킹 계산
- 상위 N개 종목 편입
- 하위권 이탈 시 교체

#### 장점

- 전략 확장성이 가장 높다.
- 향후 백테스트/최적화에 유리하다.

#### 단점

- 초기 파라미터 설계가 중요하다.

## 7. 1차 추천 기본 전략

처음 실제 구현 대상으로는 아래 순서를 추천한다.

1. `전략 5 멀티팩터 랭킹 전략`을 메인 전략으로 채택
2. `전략 1 단기 모멘텀 돌파 전략`을 보조 전략으로 추가
3. `전략 3 평균회귀 과매도 반등 전략`을 단기 테스트 전략으로 추가

이유는 다음과 같다.

- 멀티팩터 전략은 전략 선택형 구조를 만들기에 가장 적합하다.
- 점수 기반이라 종목 선별 로직을 공통 컴포넌트화하기 쉽다.
- 이후 팩터 추가/삭제가 쉬워 확장성이 좋다.
- 모멘텀 전략과 평균회귀 전략을 같이 넣으면 시장 국면별 비교가 가능하다.

## 8. 종목 유니버스 추천

초기에는 전 종목 대상보다 `필터링된 유니버스`로 시작하는 것이 안전하다.

### 추천 유니버스 조건

- KOSPI + KOSDAQ 보통주
- ETF, ETN, 스팩, 우선주 제외
- 거래정지/관리종목 제외
- 최근 20일 평균 거래대금 10억 이상
- 5일 평균 거래량 기준 미달 종목 제외
- 주가 3,000원 미만 초저가주 제외

### 이유

- 유동성 부족 종목은 모의투자에서도 체결 왜곡이 커질 수 있다.
- 저가 테마주는 신호는 좋아 보여도 자동매매 품질이 나빠질 수 있다.

## 8-1. 사용자 조건 기반 종목 리스트업 기능

자동매매와 별도로, 사용자가 원하는 조건에 맞는 종목을 장중에 리스트업하는 `조건 스캐너` 기능을 넣는 것을 권장한다.

이 기능의 목적은 다음과 같다.

- 자동매매 진입 전 후보 종목을 먼저 사람이 검토할 수 있게 함
- 특정 전략의 진입 후보군을 사전 탐색하는 용도로 사용
- 급등 가능성이 있는 종목을 `관심 후보군` 수준으로 우선 추려냄
- 텔레그램에서 바로 조회 가능하게 함

### 설계 원칙

- 외부 자료를 그대로 복제하는 방식이 아니라, 일반화된 기술적 패턴과 거래량/추세/위치 정보로 구현한다.
- 스캐너 결과는 `급등 확정 종목`이 아니라 `관심 후보군`으로 표현한다.
- 자동매매 진입 조건과 스캐너 조건은 분리하되 재사용 가능한 공통 지표를 쓴다.
- 조건식은 사용자가 수정 가능해야 한다.

### 추천 기능 구조

1. 기본 필터
- 시장, 가격, 거래대금, 거래정지 여부, 관리종목 여부

2. 기술적 조건 필터
- 최근 N일 신고가 근접
- 최근 N일 저점 대비 반등
- 거래량 급증
- 이동평균선 회복
- 변동성 수축 후 확장
- 장중 고가 돌파

3. 패턴 스코어링
- 단순 조건 통과가 아니라 패턴 점수화
- 예: 반등 점수 + 거래량 점수 + 추세 점수 + 위치 점수

4. 결과 출력
- 조건 충족 종목 리스트
- 점수순 정렬
- 조건별 통과 사유 표시
- 텔레그램/대시보드 노출

### 1차 구현 대상 패턴 스캐너

초기에는 아래 7개 패턴 스캐너를 추천한다.

#### 스캐너 1. 거래량 급증 + 가격 돌파

- 최근 20일 평균 거래량 대비 당일 거래량 급증
- 현재가가 전일 고가 또는 최근 N봉 고가 돌파
- 거래대금이 일정 수준 이상

#### 스캐너 2. 저점 반등 후보

- 단기 급락 후 1분봉/5분봉 기준 저점 형성
- 저점 이후 고점/저점이 동시에 높아짐
- 반등 구간에서 거래량 증가

#### 스캐너 3. 바닥권 횡보 후 이평선 회복

- 중기 하락 후 바닥권 박스 형성
- 5일선, 20일선 또는 분봉 기준 주요 이평 회복
- 박스 상단 재돌파 시도

#### 스캐너 4. 이중바닥 후보

- 최근 저점 구간이 2회 형성
- 두 번째 저점이 첫 번째 저점을 크게 이탈하지 않음
- 넥라인 돌파 또는 직전 반등 고점 재돌파 시도

#### 스캐너 5. 눌림목 후 재상승 후보

- 최근 상승 추세 존재
- 단기 눌림 과정에서 거래량 감소
- 재상승 시 거래량 재확대

#### 스캐너 6. 장중 랭킹 진입 후 첫 조정 후보

- 당일 상승률/거래대금 상위권 진입
- 첫 급등 후 과열 조정
- 1분봉/5분봉 기준 재돌림 후 지지 형성

#### 스캐너 7. C자리 탐색기

이 스캐너는 사용자가 제공한 PDF의 `C자리` 설명을 국내장용으로 일반화한 패턴 탐색기다.  
핵심 개념은 `강한 선행 급등 -> 과열 해소를 위한 눌림 -> 5분봉 기준 재매집 -> 재출발 가능성` 이다.

PDF에서 확인한 C자리 관련 핵심을 구현 가능한 규칙으로 정리하면 아래와 같다.

- 먼저 종목이 강하게 급등한 이력이 있어야 한다.
- 급등 직후 바로 추격하지 않고, 한 차례 눌림이 나온 뒤 다시 보는 구조다.
- 특히 강한 상승 후 장기 이평 또는 주요 저항을 돌파한 뒤 일시적으로 내려오는 구간을 중요하게 본다.
- 5분봉 기준으로 눌림 후 지지와 재반등 시도를 확인하는 것이 핵심이다.
- 바로 고점 추격이 아니라 `올랐다가 내려왔을 때`를 진입 후보로 본다.

### C자리 탐색기 판별 로직 제안

국내장 기준 1차 버전은 아래 3단계로 탐색하는 것을 권장한다.

#### 1단계. 선행 급등 종목 후보 추출

- 최근 3거래일 이내 일중 또는 일봉 기준 강한 상승 이력 존재
- 예: 최근 3거래일 최고가가 직전 20일 저점 대비 15% 이상 상승
- 최근 거래대금이 20일 평균 대비 유의미하게 증가
- 당일 또는 최근 며칠 내 상승률/거래대금 랭킹 상위 진입 이력 존재

#### 2단계. 눌림 구간 확인

- 선행 급등 이후 고점 대비 되돌림 발생
- 예: 직전 스윙 고점 대비 -20% ~ -50% 범위 조정
- 조정 중 거래량이 급등 구간 대비 감소
- 급락 붕괴가 아니라 저점이 어느 정도 유지되는 모습 확인

#### 3단계. 재출발 신호 확인

- 5분봉 기준 박스권 또는 짧은 매집 구간 형성
- 5분봉 저점이 더 이상 크게 낮아지지 않음
- 5분봉 고점/저점이 다시 높아지는 초기 신호 발생
- 5분봉 20이평 또는 60분봉 핵심 이평 회복 시 가점
- 눌림 후 첫 강한 양봉 또는 직전 단기 고점 재돌파 시 최종 후보 편입

### C자리 점수화 예시

`C자리 탐색기`는 단일 조건 통과보다 점수 기반이 더 적합하다.

- 선행 급등 점수: 최근 3일 상승률, 급등 폭, 거래대금 랭킹
- 눌림 점수: 고점 대비 조정폭, 조정 중 거래량 감소율
- 지지 점수: 5분봉 저점 유지 여부, 전저점 이탈 여부
- 재출발 점수: 5분봉 반등 강도, 단기 이평 회복, 재돌파 여부
- 위험 패널티: 거래정지 위험, 과도한 갭, 윗꼬리 과다, 연속 상한가 이후 과열

### C자리 탐색기 실전 조건 예시

```yaml
scanner_name: c_spot_rebound
market: KRX
timeframe: 5m
filters:
  min_price: 3000
  min_turnover_20d: 1000000000
  exclude_etf: true
  exclude_spac: true
conditions:
  - type: recent_surge_percent
    lookback_days: 3
    operator: ">="
    value: 15
  - type: pullback_from_recent_high
    operator: "between"
    min: 20
    max: 50
  - type: pullback_volume_ratio
    operator: "<="
    value: 0.7
  - type: intraday_box_support
    timeframe: 5m
    lookback_bars: 12
  - type: rebound_breakout
    timeframe: 5m
    lookback_bars: 6
ranking:
  - c_spot_score
  - recent_turnover_rank
  - rebound_strength
```

### C자리 탐색기에서 보는 핵심 타임프레임

- 일봉: 선행 급등, 장기이평 위치, 큰 추세 확인
- 60분봉: 추세 전환 여부와 이평 회복 확인
- 5분봉: 실제 C자리 형성 여부와 재출발 포착
- 1분봉: 너무 세밀하므로 진입 보조용으로만 사용

### C자리 탐색기와 기존 스캐너의 관계

- `장중 랭킹 진입 후 첫 조정 후보`는 C자리의 사전 후보군 역할
- `눌림목 후 재상승 후보`는 C자리의 일반화 버전
- `C자리 탐색기`는 위 두 스캐너를 더 구체적인 규칙으로 좁힌 특화 스캐너

### 텔레그램 명령 예시

- `/scan cspot` : 현재 C자리 후보 조회
- `/scan cspot top10` : 점수 상위 10개 조회
- `/scan cspot detail 005930` : 특정 종목의 C자리 점수 근거 조회

### 스캐너 조건 예시

### 자동매매와의 연결 방식

- 스캐너는 단독 조회 기능으로도 사용 가능
- 자동매매 엔진에서는 스캐너 결과를 후보 유니버스로 활용 가능
- 예: 멀티팩터 전략이 전체 시장 대신 `스캐너 통과 종목`만 대상으로 점수 계산

### 주의사항

- 급등 후보 탐색은 오탐이 많기 때문에 반드시 리스크 필터와 함께 사용한다.
- 거래량 급증만으로 진입하지 않고 추세/지지/체결 가능성을 함께 봐야 한다.
- 패턴 이름은 이해를 돕기 위한 분류이며, 실제 구현은 수치 조건과 점수화 기준으로 일반화한다.

## 10. 전략 인터페이스 설계

모든 전략은 동일한 인터페이스를 따르게 한다.  
실거래와 백테스트가 같은 전략 코드를 공유할 수 있도록 `on_bar()` 기반 구조를 정식 인터페이스로 채택한다.  
이 구조를 사용하면 실거래에서는 장중 새 분봉이 들어올 때 호출하고, 백테스트에서는 과거 분봉을 순차 재생하면서 동일하게 호출할 수 있다.

```python
class Strategy:
    name: str
    timeframe: str          # "1m", "3m", "5m", "10m", "15m"
    warmup_bars: int        # 지표 계산에 필요한 최소 과거 봉 수
    max_positions: int
    supports_overnight: bool

    def prepare_features(self, bars):
        """지표 계산 및 피처 생성"""
        ...

    def on_bar(self, current_time, data_slice, portfolio, account_state):
        """새로운 봉 도착 시 호출. 시그널을 반환한다."""
        ...

    def generate_orders(self, signals, portfolio, account_state):
        """시그널 기반으로 실제 주문을 생성한다."""
        ...
```

전략을 교체해도 실행 엔진은 그대로 재사용할 수 있다.

## 11. API 연동 설계

### 핵심 API 사용 흐름

#### 1. 인증

- 잔고 동기화
- 성과 저장

## 12. 실행 시나리오

### 거래일 캘린더 관리

- 공휴일, 임시휴장일, 선거일 등 비거래일 캘린더를 사전 관리한다.
- 반일 거래일(12:30 마감) 발생 시 장 마감 시각을 동적으로 조정해야 한다.
- 장전 동시호가(08:30~09:00), 장후 시간외 거래(15:40~16:00) 처리 여부를 전략별로 정의한다.
- 거래일 캘린더는 한국거래소 공식 일정 기준으로 연 1회 이상 업데이트한다.

### 장 시작 전

- 토큰 확보
- 계좌 상태 조회
- 종목 마스터/유니버스 준비
- 전일 기준 지표 계산
- 당일 거래일 여부 및 반일 거래일 여부 확인

### 장 초반 9:00~9:15

- 급격한 변동 구간은 관찰 위주
- 전략에 따라 초기 진입 유예

### 장중 9:15~14:30

- 5분 또는 10분 주기로 스캔
- 진입 후보 계산
- 보유 종목 손절/익절/추세 이탈 점검
- 주문 및 미체결 관리

### 장 마감 전 15:00~15:20

- 당일 청산 전략은 포지션 정리
- 오버나이트 허용 전략은 잔여 포지션 유지

### 장 마감 후

- 거래 로그 저장
- 성과 리포트 생성
- 다음 거래일 후보 준비
- 텔레그램 일간 성과리뷰 발송

### 주간/월간 리뷰 시점

- 주간 리뷰: 매주 마지막 거래일 장 종료 후 발행
- 월간 리뷰: 매월 마지막 거래일 장 종료 후 발행
- 각 리포트에는 해당 기간 동안 사용한 전략과 전략 변경 이력을 함께 표기

## 13. 포지션 sizing 제안

초기 버전은 단순하고 안정적인 방식이 좋다.

### 추천안 A. 동일가중

- 최대 5종목
- 종목당 계좌의 15% 편입
- 나머지는 현금 유지

### 추천안 B. 변동성 역가중

- ATR 또는 최근 20일 변동성 기준
- 변동성 큰 종목은 비중 축소
- 전략 성숙 후 적용 추천

### 1차 권장

`동일가중 + 종목 수 제한`으로 시작하고, 이후 역가중 방식으로 고도화한다.

## 14. 리스크 관리 상세

자동매매에서 전략보다 더 중요한 것은 리스크 통제다.

### 필수 규칙

- 1일 최대 손실 초과 시 신규 주문 중지
- 연속 손실 횟수 초과 시 전략 일시 정지
- 미체결 주문 장시간 방치 금지
- 동일 종목 중복 진입 금지
- 거래정지/VI/급등락 종목 예외 처리
- 장 마감 직전 신규 진입 제한
- 긴급 전체 청산(Kill Switch): 텔레그램 `/kill_all` 명령으로 전 포지션 시장가 청산 + 봇 즉시 정지

### 동적 포지션 축소 (드로다운 기반)

단순 중지 외에 연속 낙폭 구간에서 포지션 사이즈를 점진적으로 줄이는 동적 리스크 조절을 적용한다.

- 계좌 MDD 1% 도달 시: 종목당 최대 비중을 기본의 50%로 축소
- 계좌 MDD 2% 도달 시: 신규 진입 전면 중지, 기존 포지션만 관리
- MDD 회복 시: 단계적으로 비중 복원 (즉시 100% 복원은 금지)
- 임계치는 전략별/계좌별로 설정 가능하게 한다

### 슬리피지/체결률 실시간 모니터링

모의투자 단계부터 아래 데이터를 수집해 실전 전환 시 백테스트 가정치를 보정한다.

- 주문가 대비 실제 체결가 괴리(슬리피지) 누적 통계
- 미체결 비율: 지정가 주문 중 미체결된 비율
- 체결 소요 시간: 주문 접수 → 체결 완료까지 평균/최대 시간
- 이 데이터는 `fills` 테이블에 함께 저장하고, 성과리뷰에 요약 포함

### 모의투자 특성 반영

- 모의 계좌 체결은 실제 체결과 차이가 있을 수 있다.
- 백테스트 수익률과 모의투자 수익률 사이에 괴리가 날 수 있다.
- 따라서 체결 속도보다 `신호 품질`과 `리스크 관리`를 먼저 검증해야 한다.
- 모의투자에서는 시장가 주문이 거의 항상 즉시 체결되고, 유동성 제한이 사실상 없다.
- 실전에서는 호가 매칭에 의한 미체결, 부분체결, 슬리피지가 빈번하므로 모의투자 성과를 그대로 실전 기대치로 사용하면 안 된다.
- 모의→실전 전환 시에는 최소 4주간 모의투자 체결률/슬리피지 데이터를 수집해 실전 백테스트 가정치를 보정하는 것을 권장한다.

## 18. 백테스트/검증 방법론

단순히 전략을 만들어 돌리는 것보다 검증 절차를 분리하는 것이 중요하다.

### 현재 문서 보강 방향

백테스트는 단순한 수익률 계산기가 아니라, `분봉 기반 이벤트 재생 엔진`으로 설계하는 것을 기본 원칙으로 한다.

### 백테스트 기본 원칙

- 원천 데이터는 1분봉으로 저장한다.
- 전략 실행 단위는 전략별 타임프레임으로 분리한다.
- 일봉 지표와 분봉 지표를 함께 사용할 수 있어야 한다.
- 신호 생성 시점과 주문 체결 시점을 분리한다.
- 체결가는 이상적인 종가 체결이 아니라 보수적인 체결 규칙을 사용한다.
- 여러 전략을 동일 계좌에서 돌릴 수 있으므로 전략 단위 성과와 통합 포트폴리오 성과를 모두 계산해야 한다.

### 추천 백테스트 시간 단위

초기 구현 기준 추천 우선순위는 아래와 같다.

1. 원천 저장: `1분봉`
2. 기본 실행 주기: `5분봉`
3. 전략별 선택 가능: `3분봉`, `5분봉`, `10분봉`, `15분봉`
4. 일봉: 추세/필터용 보조 데이터

이렇게 잡는 이유는 다음과 같다.

- 1분봉만 저장해두면 대부분의 국내장 단기 전략을 유연하게 검증할 수 있다.
- 5분봉은 노이즈와 연산량 사이 균형이 좋다.
- 돌파형/반등형은 3분봉 또는 5분봉이 유리하고, 추세형은 10분봉 또는 15분봉도 적합하다.

### 백테스트 엔진 구조

#### 1. 데이터 레이어

- 종목별 1분봉 OHLCV 저장
- 거래일 캘린더 관리
- 장중 휴장/조기종료 여부 반영
- 필요 시 일봉 보조 캐시 생성

#### 1-1. 과거 데이터 초기 적재 전략

1분봉 원천 데이터를 키움 REST API로만 수집하면 종목당 수천 건의 요청이 필요하므로, 아래 방식을 병행한다.

- 야간 배치 수집: 장 종료 후 당일 1분봉 데이터를 전 종목 대상으로 순차 수집 (API Rate Limit 고려, 종목당 간격 두고 호출)
- 점진적 축적: 매일 당일 데이터를 적재하여 시간이 지남에 따라 백테스트용 데이터가 누적되는 구조
- 외부 데이터 소스 보조: 초기 과거 데이터가 부족한 구간은 일봉 기반 백테스트로 우선 검증하고, 분봉 데이터가 충분히 쌓인 후 분봉 백테스트로 전환
- 데이터 무결성 점검: 수집 후 누락 봉, 이상 OHLCV, 거래일 불일치 등을 자동 검증하는 배치 포함

#### 2. 리샘플링 레이어

- 1분봉 -> 3분/5분/10분/15분봉 변환
- 거래일 경계 기준으로 정확히 재구성
- 장 시작 직후/마감 직전 분봉 절단 처리 일관화

#### 3. 전략 실행 레이어

- 각 전략은 자신이 선언한 `timeframe` 기준으로만 바를 소비
- 동일 시점에 여러 전략이 독립적으로 시그널 생성
- 전략별 최대 보유 종목 수 및 현금 사용 한도 적용

#### 4. 주문 시뮬레이션 레이어

- 시그널 발생 시 즉시 체결이 아니라 다음 분봉 시가 또는 지정 규칙으로 체결
- 지정가 주문은 해당 분봉의 고가/저가 범위 충족 시 체결 처리
- 미체결 주문은 N개 봉 이후 취소 또는 정정 가능
- 부분체결은 2차 단계에서 추가하고, 1차는 보수적 전량체결/미체결 모델로 시작

#### 5. 포트폴리오 레이어

- 전략별 가상 계좌 분리 가능
- 통합 계좌 모드 지원
- 주문 충돌 시 우선순위 규칙 적용
- 전략별, 종목별, 전체 계좌 기준 손익 집계

### 체결 로직 권장안

자동매매 백테스트에서 가장 왜곡이 큰 부분이 체결 가정이다.  
따라서 아래처럼 보수적으로 설계하는 것이 좋다.

#### 시장가 백테스트

- 매수: 다음 봉 시가 + 슬리피지
- 매도: 다음 봉 시가 - 슬리피지

#### 지정가 백테스트

- 매수 지정가 이하로 다음 봉 저가가 내려오면 체결
- 매도 지정가 이상으로 다음 봉 고가가 올라가면 체결
- 체결 우선순위는 실제 호가를 모르므로 보수적 기준 사용

#### 슬리피지/비용

- 슬리피지: 기본 0.05% ~ 0.20% 가정
- 수수료: 모의투자와 별도로 실거래 가정 비용도 옵션화
- 거래세/제세금도 백테스트 옵션으로 분리

### 멀티전략 백테스트 방식

여러 전략 대응을 위해 아래 2가지 모드를 모두 지원하는 것이 좋다.

#### 모드 A. 전략 단독 백테스트

- 전략 하나만 선택해 독립 성과 검증
- 파라미터 튜닝과 디버깅에 적합

#### 모드 B. 전략 조합 백테스트

- 둘 이상의 전략을 동시에 실행
- 동일 계좌 자금 배분 충돌 반영
- 전략 간 상관관계와 분산효과 검증

### 멀티전략 자금 배분 예시

- 전략별 고정 비중
- 성과 기반 동적 비중
- 변동성 기반 비중
- 최대 전략 손실 한도 초과 시 해당 전략만 중지

### 분봉 백테스트에서 꼭 반영할 예외

- 시가 갭 상승/하락
- 상한가/하한가 접근으로 인한 비정상 체결
- VI 발동 구간
- 유동성 부족 종목의 체결 왜곡
- 장 마감 직전 주문 미체결
- 오버나이트 허용 전략과 당일청산 전략의 차이

### 백테스트 결과 저장 항목

- 실행 전략명
- 실행 파라미터
- 사용 타임프레임
- 거래별 진입/청산 시각
- 거래별 신호 발생 시각과 실제 체결 시각
- 거래별 손익, MFE, MAE
- 일별 손익 곡선
- 월별 성과
- 전체 성과 지표
- 기간 중 전략 변경 이력

## 19. 성과리뷰 방법론

성과리뷰는 단순 손익 요약이 아니라, "어떤 전략을 언제 사용했고 그 결과가 어땠는지"를 복기할 수 있어야 한다.

### 리뷰 단위

- 일간 리뷰
- 주간 리뷰
- 월간 리뷰

### 공통 포함 항목

- 리뷰 기간
- 기간 시작/종료 시점 계좌 평가금액
- 실현손익
- 평가손익
- 누적 수익률
- 최대낙폭
- 총 거래 수
- 승률
- 평균 손익비
- 매수/매도 횟수
- 전략별 기여 손익
- 해당 기간 활성 전략
- 해당 기간 전략 변경 이력
- 벤치마크 대비 성과: 동일 기간 KOSPI/KOSDAQ 지수 수익률, KODEX 200 ETF 수익률 대비 초과수익률

### 전략 이력 표기 방식

성과리뷰에는 반드시 아래 형태의 정보가 포함되어야 한다.

- 기간 전체 기본 전략
- 중간 전략 변경 여부
- 변경 시각
- 변경 전 전략
- 변경 후 전략
- 변경 주체

예시:

```text
전략 이력
- 2026-04-01 09:00:00 ~ 2026-04-11 10:14:00 : multi_factor
- 2026-04-11 10:15:00 ~ 현재 : pullback_trend
변경 주체: telegram
```

### 일간 리뷰 추가 항목

- 당일 신규 진입 종목
- 당일 청산 종목
- 손절/익절 발생 건수
- 장중 가장 성과가 좋았던 전략 시간대

### 주간 리뷰 추가 항목

- 요일별 손익
- 전략별 주간 기여도
- 승률 변화
- 손익 상위/하위 거래 5건

### 월간 리뷰 추가 항목

- 주차별 성과
- 전략별 월간 누적 손익
- 자금 회전율
- 시장 국면 대비 성과 코멘트

### 리포트 생성 방식

- 일간 리뷰: 매 거래일 장 종료 후 자동 생성
- 주간 리뷰: 주간 마지막 거래일 장 종료 후 자동 생성
- 월간 리뷰: 월말 마지막 거래일 장 종료 후 자동 생성
- 사용자는 텔레그램 명령으로 과거 리뷰를 재조회 가능

### 성과리뷰 전달 채널

- 1차: 텔레그램 텍스트 메시지
- 2차: 텔레그램 첨부 CSV 또는 이미지 요약
- 3차: 대시보드 상세 성과 페이지

### 검증 순서

1. 과거 일봉/분봉 데이터로 백테스트
2. 수수료/세금/슬리피지 보수적으로 반영
3. 월별/연도별 성과 확인
4. 최대낙폭(MDD), 승률, 손익비, 샤프 비율 확인
5. 모의투자에서 4주 이상 전진 검증
6. 실거래 전환은 가장 나중

### 핵심 지표

- CAGR
- MDD
- Profit Factor
- 승률
- 평균 손익비
- turnover
- 전략별 월별 손익 안정성
- 분봉 기준 체결지연 민감도
- 슬리피지 변화에 대한 성과 민감도
- 전략 조합 간 상관계수
- 전략별 장중 시간대 성과

## 20. 추천 우선순위

처음부터 너무 많은 전략을 동시에 완성하려고 하면 실패 확률이 높다.  
그래서 아래 순서를 추천한다.

1. 키움 인증, 잔고조회, 주문, 미체결 처리부터 완성
2. 일봉/분봉 적재 및 유니버스 필터 완성
3. 멀티팩터 전략 1개 먼저 실동작
4. 그 다음 모멘텀, 평균회귀 전략 추가
5. 이후 조건검색/WebSocket/대시보드 확장

## 21. 최종 권장 설계안

현재 요구사항 기준으로 가장 추천하는 1차 설계는 아래와 같다.

- 브로커: 키움 REST API
- 환경: 모의투자, 국내 KRX 종목
- 엔진: Python + FastAPI + SQLite
- 데이터 주기: 일봉 + 5분봉
- 실행 방식: 5분 주기 스캔 + 필요 주문 실행
- 기본 전략: 멀티팩터 랭킹 전략
- 보조 전략: 단기 모멘텀 돌파, 과매도 반등
- 리스크 정책: 동일가중, 최대 5종목, 일손실 제한, 종목 손절, 드로다운 기반 동적 축소
- 운영 채널: 텔레그램 봇(등급별 알림) + 내부 대시보드 + 헬스체크
- 리뷰 체계: 일/주/월 성과리뷰 + 전략 이력 + 벤치마크 대비 성과 포함
- 장애 대응: Kill Switch, 장애 복구 자동화, 상태 동기화
- 향후 확장: WebSocket 실시간 체결/시세 연동, 시장 국면 감지 기반 전략 자동 전환

## 23. 2차 확장 기능 제안

아래는 1차 구현 완료 후 고도화 단계에서 추가할 기능들이다.

### 시장 국면 감지 및 전략 자동 전환

현재는 사용자가 텔레그램으로 수동 전환하지만, 시장 국면 감지 모듈을 넣어 반자동 전환을 지원한다.

- KOSPI 지수 이동평균 배열, 등락비율, 거래대금 추이 등으로 시장 국면 판별
- 상승장 -> 모멘텀 전략 추천, 횡보장 -> 평균회귀 전략 추천, 하락장 -> 현금 비중 확대 추천
- 자동 전환은 위험하므로 "추천 + 사용자 승인" 방식으로 시작
- 텔레그램으로 "현재 시장 국면: 횡보, 추천 전략: 평균회귀 반등. 전환하시겠습니까?" 형태로 알림

### 전략 파라미터 프로파일 관리

같은 전략이라도 파라미터 세트에 따라 성과가 다르므로, 프로파일(preset) 저장/비교 기능을 넣는다.

- 전략별 파라미터 세트를 이름 붙여 저장 (예: `multi_factor_aggressive`, `multi_factor_conservative`)
- 백테스트 결과와 실전 결과를 파라미터 프로파일 단위로 추적
- 텔레그램에서 `/strategy set multi_factor --profile aggressive` 형태로 선택
